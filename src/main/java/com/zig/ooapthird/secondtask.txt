Шаг 1. Определяем границы разрабатываемой системы (анализ):

Что будет включено в систему:
•	Игровое поле 8x8 (с элементами пяти типов A, B, C, D, E);
•	Алгоритм перестановки элементов: обработка ввода пользователя, перестановка только соседних элементов;
•	Поиск и удаление набора/комбинации элементов;
•	Заполнение "пустот", сдвиг элементов, генерация новых после "исчезновений";
•	Подсчёт очков;
•	Создание и активация специальных бонусов (удаление строки/столбца, всех элементов типа и т.п.) за сложные комбинации;
•	Консольный интерфейс: вывод состояния поля, обработка хода пользователя, сообщения об ошибках, завершения игры.
Что точно не будет включено в систему:
•	Графический интерфейс: только консоль;
•	Многопользовательский режим: только одиночный режим;
•	Опция "Сохранение" (прогресс не сохраняется на диск/файл и т.п.);
•	Без БД, интернета;
•	Реклама, покупки).

Главные подсистемы
•	Система управления (отвечающая за игровой процесс);
•	Пользовательский интерфейс;
•	Система бонусов;
•	Система подсчёта очков.

Пользовательские метафоры
•	"Игровое поле" — "сетка" 8х8 для размещения элементов;
•	"Элемент" — базовая ячейка поля (A, B, C, D, E);
•	"Комбинация" — ряд/столбец из трёх и более одинаковых элементов;
•	"Бонус" — особая возможность/награда, за сложную комбинацию;
•	"Очки" — числовой показатель "успеха" игрока;
•	"Ход" — перестановка двух соседних элементов.

Функциональность
•	Заполнение игрового поля без начальных совпадений;
•	Ввод "хода" пользователем для перестановки элементов;
•	Обработка хода: проверка валидности хода, поиск комбинаций (удаление новых совпадений после исчезновения);
•	Бонусы за "сложные" комбинации;
•	Механизм активации бонусов, их виды (ряд, столбец, все А и т.п.); ?
•	Подсчёт очков;
•	История ходов и вывод статистики;
•	Обработка "ошибочных" ходов, ошибок ввода;
•	Выход по команде пользователя или если нет ходов.

Библиотеки повторного использования
•	Стандартные библиотеки языка: массивы, Random, ввода/вывода в консоль;
•	Модули/классы собственной реализации.
