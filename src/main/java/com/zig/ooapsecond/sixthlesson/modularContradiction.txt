1. Существуют ли ситуации, когда связи между модулями должны делаться публичными?

По идее это является нарушением инкапсуляции. Но на практике приходиться использовать публичные связи между модулями.
Допустим, если функционал модуля используется во многих других модулях. Первое приходит на ум - логирование.

2. Какие метрики вы бы предложили для количественной оценки принципов организации модулей?

К примеру можно следуя принципам "правильного" проектирования стараться делать как можно меньше связей.
Количество связей использовать как метрику.

3. Если вы разрабатывали программы, в которых было хотя бы 3-5 классов, как бы вы оценили их модульность по этим метрикам?

На текущем проекте есть сервис, который занимается сбором данных и отправкой,
далее приемом обработанных данных с парсингом и сохранением.
Есть интерфейс для маппинга и много реализаций для разных блоков данных.
Так же есть интерфейс парсинга с множеством реализаций. И сервис сброса и сохранения обработанных данных.
На мой взгляд можно выделить 3 модуля:
- модуль Mapping;
- модуль Parsing;
- модуль Saver;
